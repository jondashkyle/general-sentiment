# Agent Guidelines

## Content files

Files under `content/` are the user's prose and are synced from an external
Obsidian vault via `hyalite-sync`. **Never modify the text or frontmatter of
files in `content/`.** If a page needs a new field, either:

- ask the user to add it in their Obsidian source, or
- add the data to a mapping file under `src/data/` and resolve it in the
  page template (see `src/data/contributors.json`).

## Data files

`src/data/` holds generated or hand-maintained JSON used at build time:

- `releases.json` — regenerated by `npm run sync:releases` (fetches from
  the GitHub Releases API).
- `contributors.json` — hand-maintained lookup keyed by the contributor
  slug used in content frontmatter (e.g. `/jon-kyle`).

## Dates

Render dates as `YYYY-MM-DD`.
